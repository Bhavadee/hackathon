# Cprime TaaS

AI-agent-powered training operations prototype built with Next.js, PostgreSQL, pgvector, raw parameterized SQL, and Docker Compose.

## What is included

- Training operations command center
- Course freshness and certification-watchdog view
- CRM-style demand and profitability pipeline
- Trainer capacity and skill matching data model
- Runnable agent-orchestration demo
- PostgreSQL relational schema
- `pgvector` columns and HNSW cosine-search indexes for course chunks and trainer profiles
- Seeded hackathon data
- Dockerized production build

## Run everything with Docker

Make sure Docker Desktop is running, then:

```bash
docker compose up --build
```

Open [http://localhost:3000](http://localhost:3000). PostgreSQL is exposed on port `5432` for local inspection.

To stop the stack:

```bash
docker compose down
```

The database volume is preserved between runs. To intentionally recreate the seeded database, remove the volume with `docker compose down -v` and start again.

## Local development

Start only PostgreSQL in Docker:

```bash
docker compose up db -d
copy .env.example .env.local
npm run dev
```

The app falls back to clearly marked demo data when PostgreSQL is unavailable, so UI work can continue without the database container.

## Database and vector design

The reusable `pg.Pool`, query helper, and transaction wrapper live in `src/lib/pg.ts`. The Docker initialization SQL lives in `db/init/001_init.sql` and enables `pgvector`, creates the tables, adds HNSW indexes, and loads demo records. Application queries use placeholders such as `$1` rather than interpolating user input.

Raw vector similarity search is implemented in `src/lib/vector-search.ts` using pgvector's cosine-distance operator.

Embeddings use 1,536 dimensions by default, matching common small embedding models. Store only embeddings generated by the same model in an index. The next implementation step is document ingestion: extract text, chunk it, generate embeddings, and insert rows into `knowledge_chunks`.

## Environment

| Variable | Purpose |
| --- | --- |
| `DATABASE_URL` | PostgreSQL connection string |
| `OPENAI_API_KEY` | Server-side key used for live course generation |
| `OPENAI_MODEL` | Course-generation model; defaults to `gpt-5.6` |
| `EMBEDDING_MODEL` | Embedding model identifier |
| `PGSSL` | Set to `true` only when the PostgreSQL provider requires SSL |

Do not commit real API keys. Course generation calls the OpenAI Responses API from the server and returns a Zod-validated structured blueprint. The API key is never exposed to browser code.
